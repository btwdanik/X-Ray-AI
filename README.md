# X-Ray AI: классификация рентгеновских снимков грудной клетки

Всем привет! Данный проект представляет из себя AI-систему для анализа рентгеновских снимков грудной клетки человека.  
В основе решения лежит предобученная `DenseNet121`, которая предсказывает вероятности для `15` классов из датасета `VinBigData`, а результат дополнительно объясняется через `Grad-CAM`.

То есть это не только ноутбук с обучением, но и небольшое web-приложение:

1. `FastAPI` backend принимает снимок и запускает модель
2. `DenseNet121` возвращает вероятности патологий
3. `Grad-CAM` показывает, какие области сильнее повлияли на прогноз
4. frontend отображает снимок, топ-5 классов и `heatmap`

> [!CAUTION]
> Это демонстрационный учебно-исследовательский проект, а не медицинское изделие. Результаты модели нельзя использовать для постановки диагноза или замены врача-рентгенолога.

## Запуск

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите backend из корня проекта:

```bash
uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:8000
```

## Идея проекта

Основная идея такая:

> взять предобученную `DenseNet121`, заменить последний классификатор на 15 выходов и дообучить модель под multilabel-классификацию рентгеновских снимков.

Почему `multilabel`: на одном снимке может быть несколько признаков одновременно. Поэтому модель не использует `softmax`, а выдает 15 независимых логитов, которые на инференсе превращаются в вероятности через `sigmoid`.

Классы модели:

```text
Aortic enlargement - Расширение аорты
Atelectasis - Ателектаз
Calcification - Кальцификация
Cardiomegaly - Кардиомегалия
Consolidation - Консолидация легочной ткани
ILD - Интерстициальное заболевание легких
Infiltration - Инфильтрация
Lung Opacity - Затемнение легкого
Nodule/Mass - Узел или объемное образование
Other lesion - Другое поражение
Pleural effusion - Плевральный выпот
Pleural thickening - Утолщение плевры
Pneumothorax - Пневмоторакс
Pulmonary fibrosis - Легочный фиброз
No finding - Без атологических находок
```

## Обучение и результат

Обучение я проводил на платформе `Kaggle` на датасете `VinBigData Chest X-ray Abnormalities Detection`.  
Из исходной разметки была собрана multilabel-таблица: каждому снимку соответствует бинарный вектор из 15 признаков.

Данные были разделены так:

| Часть | Количество |
|---|---:|
| Train | `12 000` |
| Validation | `3 000` |

Основная конфигурация:

| Параметр | Значение |
|---|---:|
| Архитектура | `DenseNet121` |
| Размер изображения | `512x512` |
| Количество классов | `15` |
| Loss | `BCEWithLogitsLoss` |
| Optimizer | `AdamW` |
| Эпохи | `8` |
| Метрика | `mean ROC-AUC` |

Для дисбаланса классов использовался `pos_weight`, чтобы редкие патологии не терялись на фоне более частых классов.

Лучший результат сохраненного запуска:

| Метрика | Значение |
|---|---:|
| Лучшая эпоха | `8` |
| Train loss | `0.3211` |
| Validation loss | `0.4102` |
| Mean ROC-AUC | `0.9537` |

Итоговая модель лежит здесь:

```text
backend/model/densenet121_xray_512_best.pth
```

## Инференс и API

Backend принимает `JPG` или `PNG`, приводит изображение к `RGB`, ресайзит до `512x512`, применяет ImageNet-нормализацию и запускает модель.  
После этого logits переводятся в вероятности через `sigmoid`, выбирается топ-5 классов, а для самого вероятного класса строится `Grad-CAM`.

Основной endpoint:

```text
POST /predict
```

Он принимает файл в multipart-поле `file` и возвращает:

- `image` - исходное изображение в base64
- `heatmap` - Grad-CAM heatmap в base64
- `predictions` - топ-5 классов с вероятностями

`Grad-CAM` здесь используется как примерная визуализация внимания модели, а не как точная сегментация патологической области.

---

## Ограничения

- поддерживаются только `JPG` и `PNG`
- `DICOM` не поддерживается
- модель не проходила клиническую валидацию
- вероятности не являются медицинскими диагностическими вероятностями
- `Grad-CAM` не является точной медицинской разметкой

## Вывод

В итоге получился полный ML-pipeline: подготовка данных, обучение `DenseNet121`, backend на `FastAPI`, простой frontend и визуальное объяснение результата через `Grad-CAM`.

Самая важная часть проекта для меня была не просто обучить модель, а довести ее до удобного инференса: загрузил снимок, получил топ-5 вероятных классов и сразу увидел heatmap поверх изображения.
