# X-Ray AI: классификация рентгенограмм с DenseNet121

Всем привет! Данный проект представляет из себя небольшую AI-систему для анализа рентгеновских снимков грудной клетки.  
В основе решения лежит `DenseNet121`, который предсказывает вероятности для 15 классов из датасета VinBigData, а для визуального объяснения результата я добавил `Grad-CAM`.

То есть проект состоит не только из обученной модели, но и из полноценного мини-приложения:

1. `FastAPI` backend принимает снимок и запускает модель
2. `DenseNet121` возвращает вероятности патологий
3. `Grad-CAM` подсвечивает область, которая сильнее всего повлияла на прогноз
4. простой frontend показывает снимок, top-5 классов и тепловую карту

> [!CAUTION]
> Это демонстрационный ML-проект, а не медицинское изделие. Результаты модели нельзя использовать для постановки диагноза или замены врача-рентгенолога.

## Идея проекта

Основная идея довольно простая:

> взять предобученную `DenseNet121`, заменить последний классификатор на 15 выходов и дообучить модель под multilabel-классификацию рентгеновских снимков.

Почему multilabel: на одном снимке может быть не один диагноз, а сразу несколько признаков. Поэтому здесь не используется `softmax`, где классы конкурируют друг с другом.  
Вместо этого модель выдает 15 независимых логитов, а на inference к ним применяется `sigmoid`.

Классы:

```text
Aortic enlargement, Atelectasis, Calcification, Cardiomegaly,
Consolidation, ILD, Infiltration, Lung Opacity, Nodule/Mass,
Other lesion, Pleural effusion, Pleural thickening, Pneumothorax,
Pulmonary fibrosis, No finding
```

## Как я обучал модель

Обучение я проводил в `Kaggle Notebook`, потому что там удобно работать с датасетом и GPU.  
Использовался датасет `VinBigData Chest X-ray Abnormalities Detection`:

- изображения: `/kaggle/input/datasets/xhlulu/vinbigdata/train`
- разметка: `/kaggle/input/competitions/vinbigdata-chest-xray-abnormalities-detection/train.csv`

В ноутбуке я сначала проверил, что `image_id` из `train.csv` корректно совпадают с PNG-файлами.  
В сохраненном запуске получилось:

- всего изображений: `15 000`
- найдено PNG: `15 000`
- пропущенных изображений: `0`
- пересечение train/validation: `0`

Дальше я собрал multilabel-таблицу: для каждого снимка сформировал вектор из 15 бинарных признаков.  
После этого разделил данные на train и validation:

| Часть | Количество |
|---|---:|
| Train | `12 000` |
| Validation | `3 000` |

Разбиение делалось в пропорции `80/20`, со стратификацией по классу `No finding`, чтобы train и validation были похожи по балансу нормальных снимков.

## Параметры обучения

Основная конфигурация обучения:

| Параметр | Значение |
|---|---:|
| Архитектура | `DenseNet121` |
| Предобученные веса | `IMAGENET1K_V1` |
| Размер изображения | `512x512` |
| Количество классов | `15` |
| Тип задачи | `multilabel classification` |
| Batch size | `64` |
| Эпохи | `8` |
| Optimizer | `AdamW` |
| Learning rate | `1e-4` |
| Weight decay | `1e-4` |
| Loss | `BCEWithLogitsLoss` |
| Scheduler | `ReduceLROnPlateau` |
| Scheduler mode | `max` |
| Scheduler factor | `0.5` |
| Scheduler patience | `1` |
| Seed | `42` |
| Workers | `2` |
| Mixed precision | `torch.cuda.amp` при наличии CUDA |
| Multi-GPU | `nn.DataParallel`, если доступно несколько GPU |

Аугментации для train:

```python
T.Resize((512, 512))
T.RandomHorizontalFlip(p=0.5)
T.RandomRotation(degrees=5)
T.ToTensor()
T.Normalize(mean=[0.485, 0.456, 0.406],
            std=[0.229, 0.224, 0.225])
```

Для validation аугментаций уже нет, только resize, перевод в tensor и ImageNet-нормализация.

Отдельно я учел дисбаланс классов. Для каждого класса считался `pos_weight`:

```text
negative_count / positive_count
```

Затем вес ограничивался диапазоном от `1.0` до `20.0` и передавался в `BCEWithLogitsLoss`. Это важно, потому что редкие патологии иначе слишком легко теряются на фоне более частых классов.

## Как работает inference

В backend используется такой pipeline:

1. Пользователь загружает `JPG` или `PNG`.
2. FastAPI читает изображение через `Pillow` и переводит его в `RGB`.
3. Изображение ресайзится до `512x512`.
4. Применяется ImageNet-нормализация.
5. `DenseNet121` выдает 15 логитов.
6. Через `sigmoid` логиты превращаются в вероятности.
7. Backend возвращает top-5 классов.
8. Для самого вероятного класса строится `Grad-CAM` по слою `model.features.denseblock4`.
9. Frontend показывает исходный снимок, вероятности и heatmap.

Grad-CAM нужен не как точная сегментация, а как примерная визуализация того, куда модель "смотрела" при принятии решения.

## API

### `GET /`

Возвращает основной HTML-интерфейс.

### `POST /predict`

Принимает файл в multipart-поле `file`.

Пример:

```bash
curl -X POST "http://127.0.0.1:8000/predict" \
  -F "file=@xray.png"
```

Пример ответа:

```json
{
  "image": "data:image/png;base64,...",
  "heatmap": "data:image/png;base64,...",
  "predictions": [
    {
      "class_idx": 3,
      "name": "Cardiomegaly",
      "probability": 87.42
    }
  ]
}
```

Фактически `predictions` содержит до 5 элементов, отсортированных по убыванию вероятности.

## Ограничения

- Поддерживаются только `JPG` и `PNG`.
- `DICOM` не поддерживается.
- Изображение принудительно приводится к квадрату `512x512`.
- Модель не проходила клиническую валидацию.
- Вероятности не являются медицинскими диагностическими вероятностями.
- Grad-CAM не является точной разметкой патологической области.
- В приложении нет авторизации, базы данных и истории исследований.
